## üèóÔ∏è Architecture & State

### 1. **Real-time Synchronization**
The application uses the `firebase/database` SDK with custom React Hooks to maintain a low-latency connection:
- `useRealtimeWorkers.js`: Streams all worker biometric data.
- `useRealtimeMachines.js`: Streams machine telemetry and PdM predictions.
- `useRealtimeAlerts.js`: Listens for actionable recommendations generated by the backend.
- `useRealtimeRestRequests.js`: Tracks operator-initiated break requests.

### 2. **Component Structure**
-   `GlobalHeader`: Real-time cumulative fleet health (Critical/Warning/Safe counts).
-   `WorkerCard`: Compact, animated summary of worker bio-states and assigned machine.
-   `ActionableAlerts`: Dedicated triage panel for incoming risk event analysis.
-   `RestRequestPanel`: Workflow management for approving/denying operator breaks.

## üß† Functional Logic: CIS Visualizer

The dashboard re-calculates the **Composite Intelligence Score (CIS)** in real-time to provide visual redundancy to the backend logic.
- **Normalization Engine**: Converts raw sensor data (BPM, ms, ¬∞C) into linear risk gradients (0-1).
- **Calibrated Thresholds**: Aligned with the `Worker` and `Machine` profiles defined in `backend/models.py`.
